# 快速开始指南

## 🚀 训练脚本启动方式

### 方式一：快速训练（推荐）

最简单快捷的启动方式，直接使用环境中的Python：

```bash
bash scripts/training/快速训练.sh
```

### 方式二：完整训练命令

包含环境激活和完整配置的训练启动：

```bash
bash scripts/training/完整训练命令.sh
```

### 方式三：GPU训练启动

GPU训练专用启动脚本：

```bash
bash scripts/training/GPU训练启动脚本.sh
```

### 方式四：基线训练

运行基线实验：

```bash
# CPU基线
bash scripts/training/run_baseline_cpu.sh

# nuScenes基线
bash scripts/training/run_baseline_nuscenes.sh
```

## 📊 监控脚本启动方式

### 方式一：实时监控训练（推荐）

最详细的监控信息，显示训练进度、loss、学习率、GPU使用率等：

```bash
bash scripts/monitoring/实时监控训练.sh
```

### 方式二：使用watch命令自动刷新

简洁版监控，使用系统watch命令每2秒刷新一次：

```bash
watch -n 2 -c bash scripts/monitoring/简洁监控训练.sh
```

或者：

```bash
# 使用启动脚本（自动检测watch命令）
bash scripts/monitoring/启动实时监控.sh
```

### 方式三：直接运行版（自动刷新）

内置自动刷新功能的监控脚本，无需额外命令：

```bash
# 使用默认刷新间隔（2秒）
bash scripts/monitoring/实时监控（直接运行版）.sh

# 指定刷新间隔（例如5秒）
bash scripts/monitoring/实时监控（直接运行版）.sh 5
```

### 方式四：简洁监控

基础监控信息，快速查看：

```bash
bash scripts/monitoring/简洁监控训练.sh
```

### 方式五：基础监控

最简单的监控脚本：

```bash
bash scripts/monitoring/监控训练.sh
```

## 🛠️ 其他实用脚本

### 诊断训练进程

如果监控脚本无法检测到训练进程，可以使用诊断脚本：

```bash
bash scripts/utils/诊断训练进程.sh
```

### 检查环境

快速检查训练环境配置：

```bash
python scripts/utils/check_environment.py
```

### 准备数据

准备nuScenes BEV数据：

```bash
bash scripts/data/prepare_nuscenes_bev_data.sh
```

### 清理临时文件

清理评估产生的临时文件：

```bash
bash scripts/data/清理临时文件.sh
```

## 📝 脚本路径说明

所有脚本均已整理到 `scripts/` 目录下，按功能分类：

- **训练脚本**: `scripts/training/`
- **监控脚本**: `scripts/monitoring/`
- **环境配置脚本**: `scripts/setup/`
- **数据准备脚本**: `scripts/data/`
- **工具脚本**: `scripts/utils/`

## 💡 使用建议

1. **首次训练**: 使用 `快速训练.sh` 或 `完整训练命令.sh`
2. **监控训练**: 推荐使用 `实时监控训练.sh` 获取详细信息
3. **后台训练**: 使用 `nohup` 或 `screen`/`tmux` 配合监控脚本
4. **问题诊断**: 使用 `诊断训练进程.sh` 排查问题
