# 监控脚本快速启动指南

## ✅ 最简单的启动方式

### 方式1: 直接运行（推荐）

```bash
cd /media/ldk950413/data0/DiffBEV
bash scripts/monitoring/实时监控训练.sh
```

**说明**: 这会显示一次监控信息。如果需要自动刷新，使用方式2。

### 方式2: 自动刷新版本

```bash
cd /media/ldk950413/data0/DiffBEV
bash scripts/monitoring/实时监控（直接运行版）.sh
```

**说明**: 这个版本会自动刷新（默认2秒），按 Ctrl+C 停止。

### 方式3: 使用启动脚本（最推荐）

```bash
cd /media/ldk950413/data0/DiffBEV
bash scripts/monitoring/启动实时监控.sh
```

**说明**: 
- 如果系统有 `watch` 命令，会自动使用
- 如果没有，会使用内置循环模式
- 默认每2秒刷新一次
- 可以指定刷新间隔：`bash scripts/monitoring/启动实时监控.sh 5` (5秒刷新)

## 🔍 如果无法启动

### 检查1: 确认脚本存在

```bash
ls -l scripts/monitoring/*.sh
```

### 检查2: 确认在正确目录

```bash
pwd
# 应该显示: /media/ldk950413/data0/DiffBEV
```

### 检查3: 检查脚本权限

```bash
chmod +x scripts/monitoring/*.sh
```

### 检查4: 查看错误信息

```bash
bash scripts/monitoring/实时监控训练.sh 2>&1
```

## 💡 常见问题

### 问题1: "找不到训练进程"

**原因**: 训练可能还没启动或已结束

**解决**: 
```bash
# 检查训练是否在运行
ps aux | grep train.py
```

### 问题2: "路径错误"

**原因**: 不在项目根目录

**解决**: 
```bash
cd /media/ldk950413/data0/DiffBEV
bash scripts/monitoring/实时监控训练.sh
```

### 问题3: watch命令格式混乱

**原因**: watch与某些颜色输出不兼容

**解决**: 使用直接运行版：
```bash
bash scripts/monitoring/实时监控（直接运行版）.sh
```
